# Guardrails — error-born rules (Anti-Fragility Loop)
# Format: - [G-NNN] <imperative rule> — cause: <what broke> (YYYY-MM-DD)
- [G-011] A `'use server'` file may ONLY export async functions — never co-locate a plain `export const` (cookie names, URLs, other constants) in the same module; put shared constants in a plain (non-directive) file like config.ts instead — cause: `WHOOP_STATE_COOKIE` exported from actions.ts broke the Next.js build ("Only async functions are allowed to be exported in a 'use server' file"), caught by in-browser verification, not `tsc`/vitest (2026-07-03)
- [G-010] Run Python scripts that print non-ASCII (accents, Δ, ⓘ) with PYTHONIOENCODING=utf-8 on Windows — the default cp1252 console codec raises UnicodeEncodeError AT PRINT TIME, and a summary print inside finally crashing masks the real result (suite looked like it produced no output for two runs) — cause: E2E results print died on 'Δ carga' in a test name (2026-07-02)
- [G-009] Repos that format-check line endings (Biome/Prettier default LF) MUST commit a .gitattributes with `* text=auto eol=lf` — on Windows core.autocrlf=true smudges checkouts to CRLF, so `verify` greenness depends on HOW files reached disk, not on committed content; if a format check fails on files with no git diff, suspect line endings first — cause: session-8 verify opened RED (4 format errors incl. 2 committed-clean files) after some git operation re-smudged them; `git diff` showed nothing because autocrlf normalizes the comparison (2026-07-02)
- [G-008] In the Bash tool on Windows, ALWAYS write paths with forward slashes — inside double quotes a backslash before $VAR escapes the interpolation ("plans\$f" tests the literal path "plans$f"), so builtin tests (test -f, [[ ]]) return silent false negatives; re-verify any "file missing" result with forward slashes before reporting it — cause: /lint-knowledge inaugural run flagged two plan memos as MISSING that exist (one had been Read minutes earlier) (2026-07-02)
- [G-007] When seeding a Supabase auth user directly via SQL (no dashboard/admin API), set ALL of GoTrue's token/change varchar columns to '' (confirmation_token, recovery_token, email_change, email_change_token_new, email_change_token_current, phone_change, phone_change_token, reauthentication_token) — leaving them NULL makes GoTrue crash scanning NULL into a Go string, so signInWithPassword returns AuthRetryableFetchError 500; also insert the matching auth.identities row (provider='email', provider_id=user_id::text, identity_data={sub,email,...}) and note identities.email is GENERATED (never insert it) — cause: hand-crafted auth.users insert for the demo user signed-in 500 until token columns were coalesced to '' (2026-07-02)
- [G-006] NEVER round-trip source files through PowerShell 5.1 Get-Content/Set-Content for edits — its default read encoding is ANSI, so BOM-less UTF-8 accents mojibake silently (and -Encoding utf8 on write adds a BOM); use the Edit tool for source, and if shell rewriting is unavoidable use [IO.File]::ReadAllText/WriteAllText with explicit UTF8 — cause: batch regex reskin turned "síntesis/hipótesis/todavía" into "sÃ­ntesis/hipÃ³tesis" across 3 files (2026-07-01)
- [G-005] When killing a port owner from netstat output, filter to LISTENING rows FIRST (`Select-String ':PORT.*LISTENING'`) — TIME_WAIT rows carry PID 0/4 (system) and feeding them to taskkill attempts to kill critical system processes — cause: G-002 cleanup one-liner parsed all :3000 rows and taskkill'd PID 0/4 (denied by Windows, no harm) (2026-07-01)
- [G-000] HARD BAN: the Supabase project "yogastasis" is OFF-LIMITS — never call ANY tool (MCP/CLI/SQL/dashboard) against its project id, never read its config, logs, tables, or keys. All Supabase operations MUST target the athlete-os project id explicitly — cause: standing owner instruction, not error-born (2026-07-01)
- [G-004] E2E scripts must be idempotent across re-runs: select buttons whose label changes with persisted state via regex/role (e.g. "Guardar|Actualizar"), never a single literal — cause: 2nd run found "Actualizar ✓" where run 1 left "Guardar check-in", click timed out 30s (2026-07-01)
- [G-003] Keep vitest.config resolve.alias in sync with tsconfig paths — Vite does NOT read tsconfig "paths"; any source file imported via `@/` breaks test loading even when tsc is green (candidate /scaffold hardening, seen_in: athlete-os) — cause: engine tests failed to LOAD (not assert) on `@/lib/dates` (2026-07-01)
- [G-002] Before launching a dev server for E2E, verify the target port actually RESPONDS (curl) or is free — on Windows, with_server.py's stop kills the pnpm wrapper but LEAVES the node child alive holding the port half-dead; find the owner (netstat -ano | grep :PORT) and `taskkill /F /T /PID <pid>` before rerunning — cause: zombie node (PID 764) held :3000, page.goto timed out 90s while the helper reported "server ready" (2026-07-01)
- [G-001] After ANY config auto-migration (e.g. `biome migrate`), diff the config and re-verify semantics with a known-bad probe file (the check must EXIT non-zero — a printed warning is NOT enough, warnings don't fail `biome check`) before trusting a green check — cause: biome migrate 2.5.1 rewrote rules `"recommended": true` → `"preset": "none"`, silently disabling all lints while `verify` stayed green (2026-07-01)
